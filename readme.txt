1. Подготовка среды (для Windows)
1.1 Предполагается, что Git установлен  https://git-scm.com/downloads
1.2 Установить PHP версии не ниже 7.0  https://www.php.net/manual/ru/install.windows.php
1.3 Установить менеджер зависимостей Composer https://getcomposer.org/doc/00-intro.md#installation-windows
Не забудьте создать composer.bat согласно инструкции.
1.4 Убедиться, что в системной переменной PATH содержатся пути к исполняемым файлам php.exe, composer.exe, git.exe
1.5. Перезагрузить компьютер

2. Подготовка тестового проекта
2.1 Перейти в директорию, где будет создаваться папка проекта
2.2 Скопировать в нее приложенный файл deploy.bat и запустить
Ожидаемый результат: 
- создается папка todo-codecept
- в нее клонируется проект https://github.com/nimestel/todo-codecept
- подтянутся последние изменения из репозитория
- установятся пакеты, необходимые для работы тестов

3. Запуск тестов
3.1 Запустить файл start_selenium.bat
Не закрывать консоль!
Ожидаемый результат: откроется консоль с трассировкой команд selenium server, который необходим для работы тестов

3.2 Запуск тестов для десктопной версии
3.2.1 Запустить файл run_tests_desktop.bat
Ожидаемый результат:
- запустится браузер chrome с прохождением автотестов на разрешении экрана 1366x768
- затем запустится браузер firefox с прохождением автотестов на разрешении экрана 1366x768
- в консоли отображается ход выполнения тестов
- по окончанию прохождения в браузере откроется html-отчет о тестировании со скриншотами и сообщениями о найденных ошибках из директории /tests/_output/report.html 

3.3 Запуск тестов для мобильной версии
3.3.1 Запустить файл run_tests_mobile.bat
Ожидаемый результат:
- запустится браузер chrome с прохождением автотестов на разрешении экрана 360x640
- затем запустится браузер firefox с прохождением автотестов на разрешении экрана 360x640
- в консоли отображается ход выполнения тестов
- по окончанию прохождения в браузере откроется html-отчет о тестировании со скриншотами и сообщениями о найденных ошибках из директории /tests/_output/report.html 

3.4 Если запустить тесты все же не удалось, примеры отчетов можно посмотреть в папке /report_examples.
